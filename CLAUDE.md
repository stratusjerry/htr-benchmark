This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Purpose

Python framework for benchmarking multiple AI vision models on handwritten text recognition (HTR) from PDF samples. Compares model transcription accuracy using CER (Character Error Rate) and WER (Word Error Rate).

Reference article: https://generativehistory.substack.com/p/has-google-quietly-solved-two-of

## Commands

```bash
# Setup
uv sync --extra dev
cp .env.example .env            # Add API keys

# Step 1: Generate ground truth (Gemini transcribes all PDFs in files/)
uv run python run_benchmark.py --generate-ground-truth

# Step 2: Run benchmarks against ground truth
uv run python run_benchmark.py                                     # All models
uv run python run_benchmark.py --models "gemma-3-12b"              # Single model
uv run python run_benchmark.py --batch                             # Gemini batch API (cheaper)
uv run python run_benchmark.py --list-models                       # Show available models

# Tests
uv run pytest tests/ -v
uv run pytest tests/test_evaluate.py -v                            # Single test file
```

## Architecture

```
run_benchmark.py                  # CLI entry point (argparse)
htr_benchmark/
├── config.py                     # Model registry, HTR prompt, .env loading
├── pdf_converter.py              # PDF → base64 PNG images (pymupdf/fitz)
├── models/
│   ├── base.py                   # Abstract HTRModel base class
│   ├── gemini.py                 # Google Gemini via google-genai (standard + batch API)
│   ├── bedrock.py                # Claude on AWS Bedrock via Bedrock API key
│   └── lmstudio.py               # OpenAI-compatible adapter for all LMStudio models
├── evaluate.py                   # CER/WER computation via jiwer
├── runner.py                     # Orchestrator: discover samples → run models → collect results
└── output.py                     # CSV/JSON file output + console summary table
```

**Data flow**: `run_benchmark.py` → `runner.run_benchmark()` → for each model: process pages sequentially (default), or with `--batch` Gemini models submit all pages as a single batch job → `evaluate()` → `output` saves CSV/JSON/transcription files.

## Supported Models

- **Gemini 3.1 Pro Preview** — baseline, via `google-genai` SDK (`GEMINI_API_KEY`); supports `--batch` for batch API
- **Gemini 3 Pro Preview** — via `google-genai` SDK; supports `--batch`
- **Claude Sonnet 4.6** — via AWS Bedrock API key (`BEDROCK_API_KEY` + `BEDROCK_BASE_URL`)
- **5 local models via LMStudio** (OpenAI-compatible API at `localhost:1234/v1`):
  Llama 3.2 11B Vision, DeepSeek-OCR-GGUF, gemma-3-12b, gemma-3-27b, olmocr-2-7b

Model registry is in `config.py`. LMStudio `model_id` values must match what LMStudio reports (check `http://localhost:1234/v1/models`).

## Ground Truth

Generated by running `--generate-ground-truth`, which uses Gemini to transcribe all PDFs and saves `.txt` files alongside them. PDFs that already have a `.txt` are skipped. Benchmarks then score other models against these Gemini baselines.

## Key Context

- LMStudio runs one model at a time — run benchmarks per-model or let the framework skip unavailable models
- Text is normalized (lowercase, whitespace collapse) before computing metrics
- Output files are timestamped to preserve results across runs
- `files/` and `output/` directories are gitignored
