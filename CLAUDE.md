## Project Purpose

Python framework for benchmarking multiple AI vision models on handwritten text recognition (HTR) from PDF samples. Compares model transcription accuracy using CER (Character Error Rate) and WER (Word Error Rate).

Reference article: https://generativehistory.substack.com/p/has-google-quietly-solved-two-of

## Commands

```bash
# Setup
uv sync --extra dev
cp .env.example .env            # Add GEMINI_API_KEY

# Step 1: Generate ground truth (Gemini transcribes all PDFs in files/)
uv run python run_benchmark.py --generate-ground-truth

# Step 2: Run benchmarks against ground truth
uv run python run_benchmark.py                                    # All models
uv run python run_benchmark.py --models "gemma-3-12b"             # Single model
uv run python run_benchmark.py --list-models                      # Show available models

# Tests
uv run pytest tests/ -v
uv run pytest tests/test_evaluate.py -v                           # Single test file
```

## Architecture

```
run_benchmark.py                  # CLI entry point (argparse)
htr_benchmark/
тФЬтФАтФА config.py                     # Model registry, HTR prompt, .env loading
тФЬтФАтФА pdf_converter.py              # PDF тЖТ base64 PNG images (pymupdf/fitz)
тФЬтФАтФА models/
тФВ   тФЬтФАтФА base.py                   # Abstract HTRModel base class
тФВ   тФЬтФАтФА gemini.py                 # Google Gemini via google-genai SDK
тФВ   тФФтФАтФА lmstudio.py              # OpenAI-compatible adapter for all LMStudio models
тФЬтФАтФА evaluate.py                   # CER/WER computation via jiwer
тФЬтФАтФА runner.py                     # Orchestrator: discover samples тЖТ run models тЖТ collect results
тФФтФАтФА output.py                     # CSV/JSON file output + console summary table
```

**Data flow**: `run_benchmark.py` тЖТ `runner.run_benchmark()` тЖТ for each model, for each PDF: `pdf_converter` тЖТ `model.transcribe()` тЖТ `evaluate()` тЖТ `output` saves CSV/JSON/transcription files.

## Supported Models

- **Google Gemini 2.0 Flash** тАФ baseline, via `google-genai` SDK with API key from `.env`
- **5 local models via LMStudio** (OpenAI-compatible API at `localhost:1234/v1`):
  Llama 3.2 11B Vision, DeepSeek-OCR-GGUF, gemma-3-12b, gemma-3-27b, olmocr-2-7b

Model registry is in `config.py`. The `model_id` values must match what LMStudio reports (check `http://localhost:1234/v1/models`).

## Ground Truth

Generated by running `--generate-ground-truth`, which uses Gemini to transcribe all PDFs and saves `.txt` files alongside them. PDFs that already have a `.txt` are skipped. Benchmarks then score other models against these Gemini baselines.

## Key Context

- LMStudio runs one model at a time тАФ run benchmarks per-model or let the framework skip unavailable models
- Text is normalized (lowercase, whitespace collapse) before computing metrics
- Output files are timestamped to preserve results across runs
- `files/` and `output/` directories are gitignored
